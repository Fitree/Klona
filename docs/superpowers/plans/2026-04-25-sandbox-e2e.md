**Goal:** Add `sandbox/run_e2e.sh`, a one-command Docker Compose E2E runner that starts an isolated test memory server container and a sandbox test container, installs KLONA for OpenCode, verifies the actual OpenCode session plugin injects the mounted mental model into a fake OpenAI-compatible model request, uninstalls KLONA, and cleans up automatically.

**Architecture:** Add a dedicated E2E Compose file with two services: `test-memory-server` built from `memory_server/` and `test-env` built from `sandbox/Dockerfile`. The host runner invokes Compose with fixed project name `sandbox`, `--abort-on-container-exit`, and `--exit-code-from test-env`; the container test script executes Scenario 1 through actual `opencode run` against a local fake provider.

**Tech Stack:** Bash, Docker Compose v2, Python stdlib `unittest`, Python stdlib JSON helpers, `curl`, existing KLONA Python installer and Dockerfiles.

- [ ] **Step 2: Create `sandbox/run_e2e.sh` with fixed project naming and cleanup-status handling**

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd -- "$(dirname -- "${BASH_SOURCE[0]}")" && pwd)"
REPO_ROOT="$(cd -- "$SCRIPT_DIR/.." && pwd)"
COMPOSE_FILE="$REPO_ROOT/sandbox/docker-compose.e2e.yml"
PROJECT_NAME="sandbox"

cleanup() {
  local status=$?
  local cleanup_status=0

  set +e
  docker compose -p "$PROJECT_NAME" -f "$COMPOSE_FILE" down -v --remove-orphans
  cleanup_status=$?
  set -e

  if [[ "$cleanup_status" -ne 0 ]]; then
    printf 'WARNING: docker compose cleanup failed with status %s\n' "$cleanup_status" >&2
  fi

  if [[ "$status" -ne 0 ]]; then
    exit "$status"
  fi

  exit "$cleanup_status"
}

trap cleanup EXIT

cd "$REPO_ROOT"
docker compose -p "$PROJECT_NAME" -f "$COMPOSE_FILE" up --build --abort-on-container-exit --exit-code-from test-env
```
